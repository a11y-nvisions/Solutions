라디오 버튼

개요(Overview)

라디오 버튼은 여러 개의 조건 중, 하나만을 선택하는 요소다. 라디오 기기에서 눌려있지 않은 버튼을 눌렀을 때, 먼저 눌려있던 버튼이 있다면 눌려있던 버튼의 고정이 풀려 올라오고, 누른 버튼이 고정되는 물리적 구조가 유례가 되어 붙혀진 요소 이름이다.

이름 그대로 90년대부터 2000년대 초까지 주변에서 쉽게 찾을 수 있던 카세트 라디오를 생각해 보거나 카세트 라디오가 잘 생각나지 않는다면 지금도 볼 수 있는 선풍기의 버튼을 떠올리면 이 요소의 근본에 대한 이해가 쉽다. 비슷한 요소로는 select(콤보박스) 요소가 있다.

핵심 개념

주요 속성과 원칙
role="radio" 접근성 API를 통해 커스텀 요소를 스크린 리더에 라디오 버튼으로 전달한다
aria-checked="bolean"

라디오 버튼이 선택되었는가를 접근성 API를 통해 스크린 리더 사용자에게 전달한다.

속성값은 다음과 같다.

true : 라디오 또는 체크박스가 선택됨을 전달한다.

false : 라디오 또는 체크박스가 선택 해제됨 상태를 전달한다

undefined : 사용하지 않는 값으로, 속성이 없는 경우 API에 전달되는 값이다.
원칙
적절한 요소유형 사용

여러 조건 중 하나를 선택할 수 있는 radio 입력서식 요소는 반드시 role="radio"를 사용하여 요소 유형을 제공해야 하며, menu의 하위 항목인 menuitem에 적용하려면 radio 유형이나 menuitem 역할 대신에 menuitemradio 역할을 사용해야 한다.
상태정보와 초점관리

disabled 상태: 모든 입력서식 요소에는 disabled 상태가 존재한다(사용 할 수 없음, 흐리게 표시됨, 사용 중지됨)

disabled 상태에 대한 가이드는 올바른 커스텀 버튼 제공방법 - 버튼의 상태 및 초점 관리를 참조

선택정보와 초점관리
  1. 상태정보 제공 규칙
    1. radio는 checkbox와 달리 여러 개를 동시 선택하는 요소가 아니므로 aria-checked의 true 값이 중복으로 사용되는 오류를 피해야 함.
    2. 선택 안된 요소라고 하여도 aria-checked의 값을 false로 상태를 명시해주어야 한다.
  2. 상태정보에 따른 초점 제공 규칙
    1. Tab 키로 초점 이동 시 라디오 버튼의 경우, 선택된 라디오 요소에만 초점이 제공되어야 한다.
    2. 만약 checked 상태속성이 기본값으로 삽입되어 있지 않은 경우, 선택할 수 있도록 첫 번째 라디오 요소에 초점을 제공해야 한다.
  3. 라디오의 그룹단위 제공
    1. 커스텀 라디오 버튼은 fieldset이나, role="radiogroup"을 사용한 컨테이너 등을 사용하여 그룹으로 묶는 것이 바람직하다.
    2. fieldset은 반드시 legend를 통해 어떠한 그룹인지 그룹 이름을 제공해주어야 하며, 마찬가지로 role="radiogroup"또한 aria-labelledby나 aria-label로 그룹명을 제공해야 한다.

키보드 조작

메뉴아이템의 라디오 버튼을 제외한 일반적인 라디오 버튼은 마우스 뿐만 아니라 키보드를 사용하여 조작이 가능해야 한다.

  1. 다음 라디오 버튼 선택 : 오른쪽 화살표 키 아래 화살표 키, 마지막 요소에서는 첫번째 요소로 순환 이동
  2. 이전 라디오 버튼 선택 : 왼쪽 화살표 키 또는 위 화살표 키, 첫 번째 요소에서는 마지막 요소로 순환 이동
  3. 기본값 누락 시 오류 처리 : 기본 선택값을 제공하는 것이 바람직하나 기본 선택값이 없을 경우에 키보드 사용자가 선택이 가능하도록 Space나 Enter를 통해 라디오 버튼을 활성화할 수 있어야 한다.

운영체제에서 제공하는 기본 라디오 버튼은 방향키로 선택하는 것이 일반적이며, 이로 인해 각 라디오 버튼마다 초점이 가는 것은 불필요한 탐색을 요구하게 된다. 따라서 위 원칙에서 설명했던 것 처럼 선택된 항목에만 초점이 제공되어야 한다.
